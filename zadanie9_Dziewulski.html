1. Jak wstawia się styl zewnętrzny do dokumentu HTML?

Styl zewnętrzny wstawiany jest do dokumentu HTML za pomocą znacznika <link>, który umieszczany jest wewnątrz sekcji <head>. Znacznik ten może występować wielokrotnie, ale zaleca się stosowanie jednej głównej deklaracji stylów.

<link rel="stylesheet" href="style.css">

2. Gdzie wstawiany jest styl zewnętrzny?

Styl zewnętrzny wstawiany jest w sekcji <head> dokumentu HTML. Odpowiedzialny za to jest znacznik <link>, który może występować wielokrotnie, ale zaleca się ograniczenie do jednej głównej deklaracji.

3. Najprostsza forma wstawiania stylu zewnętrznego

<link rel="stylesheet" href="style.css">

4. Ile arkuszy styli może być dołączonych?

Do jednego dokumentu HTML można dołączyć wiele arkuszy styli CSS. Ważne jest, aby zachować zasadę priorytetu – style zdefiniowane później mogą nadpisywać wcześniejsze.

5. Co się dzieje w przypadku konfliktów styli?

W przypadku konfliktów (gdy ten sam styl jest definiowany w różnych arkuszach), przeglądarka stosuje zasadę kaskadowości i specyficzności.

  • Style później załadowane nadpisują wcześniejsze.
  • Style o wyższej specyficzności (np. ID zamiast klasy) mają pierwszeństwo.
  • Style !important mają najwyższy priorytet.

6. Jak tworzyć i edytować pliki CSS w edytorze tekstowym?

Pliki CSS można tworzyć i edytować w dowolnym edytorze tekstowym, np. Notatnik, VS Code. Tworzymy nowy plik, zapisujemy go z rozszerzeniem .css i wpisujemy reguły CSS.

7. Jak tworzyć i edytować pliki CSS w edytorze wizualnym?

Specjalne edytory CSS, jak Adobe Dreamweaver, Brackets, umożliwiają edycję CSS z podglądem na żywo.

8. Jakie rozszerzenie ma plik CSS?

Plik zewnętrznego arkusza styli posiada rozszerzenie .css.

9. Zalety zewnętrznego arkusza styli:

  • Łatwość zarządzania – zmiany w jednym pliku wpływają na cały serwis.
  • Lepsza wydajność – pliki CSS są cache’owane, co przyspiesza ładowanie stron.

10. Uwagi na temat pisania w CSS

  • Należy stosować /* komentarze */.
  • Unikać nadmiarowego używania !important.
  • Stosować jednolitą składnię i czytelne nazwy klas.
  • Optymalizować kod poprzez unikanie zbędnych reguł.
